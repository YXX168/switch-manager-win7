# Security Policy

## Reporting a vulnerability

Please open a GitHub security advisory rather than a public issue when the report contains an exploitable vulnerability, credential exposure, or a method for bypassing read-only/admin protections.

Never attach real switch configurations, device credentials, SNMP communities, management addresses, audit logs, or the application `data` directory to an issue.

## Operational warning

This project interacts with network infrastructure. Test against a lab or non-critical device first. The maintainers cannot guarantee command compatibility across every VRP/Comware release or AAA policy.
